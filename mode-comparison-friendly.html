Claude Desktop · Windows

Three Modes, Three Jobs

The Windows desktop app splits work across three modes. Each is its own execution environment with distinct capabilities, file permissions, and memory rules.

✨ Find Your Mode

1. Do you need Claude to read/edit local files or run terminal commands?

2. Do you want Claude to work autonomously in the background, or collaborate interactively?

In a hurry? Pick by what you're doing

01

Chat

Conversation, Projects, memory, and Artifacts — no local file access.

Best for: thinking through a problem
✓
🤖
02

Cowork

A local sandboxed VM handing off multi-step tasks while you stay nearby.

Best for: hands-off multi-step work
✓
📂
03

Code

Direct local file access, terminal, and GitHub PR monitoring.

  • Native install — self-updating binary straight on Windows, no WSL needed.
  • Git Bash — used as the shell tool when Git for Windows is installed (recommended).
  • PowerShell — fallback shell tool if Git for Windows isn't installed.
  • WSL — installed and run from inside a Linux WSL terminal instead.
Best for: building & shipping code

Capability Chat Cowork Code
▸Local file access ✕ No access Chat runs in Anthropic's cloud with no access to files on your computer. ✓ Direct, scoped Cowork runs a local VM on your own machine and reads/writes real files directly — but only inside the folders you share, not your whole computer. ✓ Direct Code reads and writes files directly on your machine (or wherever it's connected) — the same as opening them yourself.
▸Terminal access ✕ None No command-line access in Chat. • Sandboxed Cowork has a terminal, but it's internal to its sandboxed VM — it can't touch your real machine's shell. ✓ Integrated ★ A real terminal is built in, running commands directly in your project environment.
▸GitHub / PRs ✕ None No GitHub integration in Chat. • Sometimes Whether Cowork touches GitHub depends on the task you give it. ✓ Full ★ Code integrates with GitHub — local or remote sessions, PR review, visual diffs, and built-in git tooling.
▸Runs where Cloud only Always runs on Anthropic's servers. Local VM Runs in an isolated sandboxed virtual machine on your own computer, not in the cloud. Anywhere Can run locally on your machine, in the cloud, or on a remote machine you SSH into.
▸Parallel sessions ✕ No One conversation at a time. ✓ Yes Multiple tasks can run at once, each in its own conversation in the sidebar, and a task can spin up sub-agents for parallel workstreams. ✓ Yes Multiple sessions can run side by side using separate Git worktrees — isolated working folders checked out from the same repo, so tasks don't collide.
▸Execution model You drive Synchronous — Claude responds turn by turn as you chat. Runs itself ★ Once started, it runs unattended in the background without you steering each step. Interactive Session-based — you and Claude work back and forth within a coding session.
▸Survives closing app • No Your conversation history is saved, but nothing keeps running after you close the app — it's not a live process. ✕ No The desktop app must stay open — closing it (or sleeping the computer) ends the session. Scheduled tasks just catch up when you're back. • With setup Survives if it's running in cloud or SSH mode via Routines — scheduled or triggered automations that keep a session going without you at the keyboard.
▸Cross-device ✓ Yes Pick up the exact same conversation from your phone or another computer. • Message in Pro/Max users can message a running session from their phone and get results back — but the desktop app still has to stay open on your computer. • Remote view ★ Remote Control lets you view and steer a running Code session from your phone or browser.
▸Projects ✓ Yes Standard Projects, same as today. ✓ Persistent Cowork workspaces persist between runs, keeping context and files in place. • Via worktrees Project continuity comes from Git worktrees rather than a built-in Projects feature.
▸Memory ✓ Yes Standard account memory. • Within projects only Memory works inside Cowork Projects, but isn't retained across standalone sessions. • Project file Memory is scoped to a CLAUDE.md file in your project rather than account-wide memory.
▸Artifacts ✓ Yes Standard in-chat Artifacts. • Output files Instead of in-chat Artifacts, Cowork produces output files you can open directly. • Live preview See a live preview of the running app instead of a static Artifact.
▸Across own modes Export to move it Artifacts stay in Chat; download or copy the content to bring it into Cowork or Code. Lands in your files Output files save to your connected folder, so Code or Chat can pick them up directly. Shared workspace Files live in your project folder — visible whether you're in Code, or referencing them from Chat or Cowork.
▸With other Claude users Share a link Send a link to the conversation or Artifact — others with Claude can view it, and duplicate it into their own account to edit. Share the workspace Grant access to the workspace, or send a link to a run's output, so a teammate can see or continue it. Share via Git Share a pull request link or an active Remote Control session so another Claude user can follow along.
▸With non-Claude users Export the file Download the Artifact as a standalone file (HTML, image, PDF, etc.) — anyone can open it, no Claude account needed. Send the files Output files are regular files — email or share them like any other document. Publish or export Push to a shared Git host or export the finished build — recipients don't need Claude to use it.

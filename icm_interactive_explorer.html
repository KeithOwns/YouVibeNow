Systems-First AI Orchestration

The Interpretable Context Methodology (ICM) replaces complex coding frameworks with a standard filesystem structure. Developed by Jake Van Clief, it treats the folder structure as the architecture itself, enabling scalable, human-auditable AI workflows.

Folders as Code

Traditional agents hide logic in Python/TS. ICM maps sequential workflows directly onto numbered folders and markdown files on your machine.

Modular Decomposition

Borrowing from Unix pipelines, ICM breaks tasks into small, specialized stages that communicate via plain text artifacts.

The 5-Layer Context Hierarchy

To ensure the AI operates within defined bounds, ICM utilizes a standardized hierarchy of files. Explore each layer below to see how context is partitioned.

Identity Layer

The global anchor of the project. This layer contains IDENTITY.md, which defines the overarching rules, brand voice, and constraints that apply to every single stage of the process.

  • Global stylistic guidelines.
  • Tone of voice (Professional, Academic, Conversational).
  • Core constraints (Word counts, restricted terminology).

Core Principles

Five engineering-grade principles that differentiate ICM from standard prompt engineering.

"Configure the factory, not the product."

01

One Stage, One Job

Stages are atomic. A researcher only researches; it never formats the final output.

02

Plain Text Interface

Communication happens via Markdown/JSON. No proprietary databases or hidden binary states.

03

Layered Loading

The agent only loads context relevant to its current folder, drastically reducing token bloat.

04

Edit Surfaces

Every intermediate file is a human-in-the-loop checkpoint. Edit any file to steer the path.

Application Suitability

Where ICM Excels

  • ✓ Sequential, multi-step content production.
  • ✓ High-stakes research requiring audit trails.
  • ✓ Workflows with intentional human review points.

Where to Avoid

  • ✗ Real-time high-concurrency user apps.
  • ✗ Massive automated branching logic.
  • ✗ Millisecond-latency requirements.

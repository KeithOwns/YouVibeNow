Stateful Prompting Architecture

The optimal local file system structure for an LLM-assisted project. Replace monolithic "chat prompts" with powerful, persistent environmental context.

📄 README.md

Human-AI Bridge
  • Purpose: The highest-level elevator pitch of the repository.
  • Contents: Project summary, goals, and setup instructions. It's the first file an LLM (and a human) reads to orient itself.

📄 AGENTS.md

Prompt Element: Identity & Constraints
  • Purpose: Establishes who the AI is and sets strict, global boundaries.
  • Contents: Persona, primary role, and non-negotiable rules (e.g., "always write plain language," "always backup files").
📚 AUTHORITY

Supported by Andrej Karpathy's "State of GPT" (emphasizing "System Prompts" to lock in persona) and the Anthropic Prompt Engineering Guide (advocating for "Role-Playing" to drastically boost output quality).

Validated by Ethan Mollick (Wharton AI Researcher), who champions treating AI as a "co-worker" rather than software, perfectly aligning with conversational "Vibe Coding" rules.

📄 CONTEXT.md

Prompt Element: Context
  • Purpose: Provides the deep background so you don't have to re-explain the project in every chat.
  • Contents: Project summary, target audience, tech stack, and definitions of "what good looks like."
📚 AUTHORITY

Aligns with the OpenAI Prompt Engineering Guide core tactic to "Provide reference text."

Backed by Google DeepMind's Long-Context Research ("Needle in a Haystack"), which proves models like Gemini can digest massive amounts of static context accurately, provided it is structured efficiently in markdown.

📄 MEMORY.md

Prompt Element: Dynamic Constraints
  • Purpose: Acts as the AI's long-term brain across different chat sessions.
  • Contents: Newly learned formatting preferences (e.g., "keep tables to 4 columns") and a log of major architectural decisions.
📚 AUTHORITY

Mirrors the core philosophy behind Retrieval-Augmented Generation (RAG) Architecture, ensuring trusted, static context is injected into the prompt before execution to reduce hallucination.

Validated by orchestration pioneers Harrison Chase (LangChain) & Jerry Liu (LlamaIndex), whose frameworks rely entirely on the distinction between transient chat context and persistent "Long-Term Memory."

📄 TASKS.md

Prompt Element: Current Task Queue
  • Purpose: Keeps the AI anchored during complex, multi-step operations.
  • Contents: A living, indented checklist ([ ], [/], [x]) of the active sprint objectives.
📚 AUTHORITY

Backed by Andrew Ng's recent work on "Agentic Workflows" (emphasizing iterative Planning over zero-shot outputs) and landmark research on Chain-of-Thought (CoT) Prompting (Wei et al., 2022).

📁 skills/

Prompt Element: Reusable Tasks & Format
  • Purpose: Turns repetitive, multi-hour jobs into single-command instructions.
  • Contents: Sub-folders containing specific SKILL.md files (e.g., a perfect sequence of prompts to draft a blog post or deploy a server).
📚 AUTHORITY

Supported by the concept of "Least-to-Most Prompting" (Zhou et al., 2022), which proves LLMs perform exponentially better when large workflows are packaged into sequential, discrete sub-problems.

Rooted in the groundbreaking "ReAct: Synergizing Reasoning and Acting" (Yao et al., 2022) framework, demonstrating that AI succeeds when it interleaves thinking with explicit tool/skill actions.

📁 Archive/

InfoSec Layer: Availability & Integrity
  • Purpose: Protects data integrity and ensures a recoverable workspace.
  • Contents: Time-stamped backups of files before the AI edits them, plus a HISTORY.md file that acts as an audit log of past chat sessions.
📚 AUTHORITY

Ensures compliance with the OWASP Top 10 for LLM Applications. By enforcing backups and audit logs, the framework inherently mitigates risks associated with LLM hallucinations, untracked modifications, and data loss.

💬 The Active Chat Prompt

Execution Layer: Task & Output Format
  • Purpose: Your actual, typed message to the AI.
  • Contents: Because the files above handle the Identity, Context, and Constraints, your chat prompt only needs to specify the immediate micro-task and how you want the answer formatted.

The "Thin Prompt" Workflow

By offloading 80% of your prompt into the file structure above, your daily prompts become incredibly lightweight and powerful. Here is what a fully synthesized execution prompt looks like in practice:

"Review the code in @context/auth-module.py. Find the bug causing the login timeout [Task]. Return the fixed code snippet as a markdown block with inline comments explaining the fix [Output Format]."

* The AI automatically combines this short instruction with your persistent Identity, Context, and Constraints to execute the task flawlessly.

📚 AUTHORITY
On formatting workflows in HTML/Markdown—Andrej Karpathy heavily champions this, shifting from standard Markdown (for simple token efficiency) to HTML for creating structured "LLM Wikis" that allow for rich semantic rendering between multi-agent systems.
